Bugs conhecidos da 0.0.1 BACKLOG

ID Bug Causa raiz Severidade Status
BUG-01 Pattern matching em funções não é exaustivo O checker não verifica se as variantes de uma função cobrem todos os casos possíveis. fn f(0) = ... sem catchall passa silenciosamente. critical TODO
BUG-02 fn sobrescreve val com mesmo nome O checker registra fn somar depois de val somar e o símbolo mais recente vence silenciosamente. Deveria ser erro de conflito de namespace. high TODO
BUG-03 Shadowing de var não respeita regra de tipo diferente var x = 10; var x = "teste" deveria ser erro (mesmo tipo ou redeclaração de var). O checker não está aplicando a regra de shadowing corretamente para var. high TODO
BUG-04 Overloading de métodos em classes não funciona fn count(self) e fn count(self, i: Int) na mesma classe causam conflito. O checker trata métodos como um único slot, sem suporte a variantes por aridade. high TODO
BUG-05 Multiline pipe insere SEMICOLON indevido O lexer insere SEMICOLON virtual após qualquer newline em tokens que podem encerrar expressão. dobrar\n|> dividir quebra porque dobrar recebe SEMICOLON antes do |>. Precisa de lookahead para suprimir SEMICOLON quando a próxima linha começa com |>. high TODO
BUG-06 Import não resolve módulos no compilador cmd/main.go compila um arquivo único sem resolver imports. O parser produz ImportDecl mas o codegen ignora. O engine do LSP tem IndexWorkspace mas o compilador CLI não usa. critical TODO
BUG-07 Enums quebrando em vários contextos Construção e match de enums falham em cenários não-triviais. Suspeita: lookup de variante por nome no codegen não encontra o enum correto quando há múltiplos enums declarados, e o destrutor de enum não está sendo associado corretamente. critical TODO
BUG-08 for-in só funciona com ranges generateForStmt em gen_control.go retorna erro explícito para qualquer iterável que não seja RangeExpr. Listas e outras coleções não têm geração de código implementada. critical TODO
BUG-09 Pattern com literal em posição diferente não cobre call fn f(a: Int, 10) e fn f(0, b: Int) — chamada f(1, 9) não bate com nenhuma variante e falha silenciosamente em runtime ao invés de dar erro de exaustividade em compile time. high TODO
M1

Simplificação de sintaxe de funções DESIGN + IMPL

M1.0
Unificação e simplificação de declaração de funções Retorno opcional · arrow body · defaults · val=fn unification
Complexidade
CONCLUÍDO
Sintaxe alvo
              // 1. Retorno opcional (Unit implícito)
              fn log(msg: String) { print(msg) }

              // 2. Body de expressão com = (já funciona parcialmente)
              fn dobrar(x: Int) = x * 2

              // 3. val = fn unificados — val é apenas um bind do mesmo fn
              val somar = fn(a: Int, b: Int) => a + b
              // equivale a:
              fn somar(a: Int, b: Int) = a + b

              // 4. Defaults de argumentos
              fn greet(nome: String, prefixo:
              String = "Olá") -> String {
              return "$(prefixo), $(nome)!"
              }
              greet("Vand") // "Olá, Vand!"
              greet("Vand", "Ei") // "Ei,
                Vand!"

Codegen corrigido para emitir void quando ReturnType == nil em declareFuncVariants e generateFuncDecl. Inferência antecipada do tipo de retorno para funções expr-body sem anotação explícita implementada em registerFuncVariants.

M2

Novo sistema de guards com when BREAKING CHANGE

M2.0
Substituir if por when em guards de funções e match Obrigatoriedade de catchall · consistência sintática
Complexidade
ALTA
              // Funções com when — catchall obrigatório
              fn classificar(x: Int) when x < 0 -> String = "negativo"
              fn classificar(x: Int) when x == 0 -> String = "zero"
              fn classificar(x: Int) -> String = "positivo" // catchall

              // ERRO — sem catchall
              fn f(x: Int) when x > 0 = x //
                compilador exige variante sem when

              // Match também usa when (mudança de if → when)
              val r = match n {
              x when x < 0 => "negativo"
              x when x == 0 => "zero"
              _ => "positivo" // catchall
              }
Tarefas de implementação
Adicionar keyword WHEN ao lexer
Adicionar WHEN ao keywords map e ao TokenType iota. WHEN deve ser incluído em CanInsertSemicolon = false (não gera SEMICOLON após newline quando é o próximo token esperado).
